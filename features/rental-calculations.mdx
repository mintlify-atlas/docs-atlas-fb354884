## Overview

Rental commissions in the Tabulador system use a simplified calculation compared to sale transactions. The agent receives a fixed **45% of the rental cost** after the 7% tax deduction.

<Note>
Unlike sale transactions, rental calculations do NOT require answering scenario questions about exclusivity, listing, or closing. The commission is always 45%.
</Note>

## Rental Commission Formula

The calculation follows these steps:

<Steps>
  <Step title="Principal Commission">
    For rentals, the principal commission equals the rental cost:
    ```typescript
    comisionPrincipalRenta = costoNum
    ```
  </Step>
  <Step title="Apply 7% Tax Deduction">
    The 7% tax is deducted from the principal:
    ```typescript
    comisionNetaRenta = comisionPrincipalRenta * 0.93
    ```
  </Step>
  <Step title="Calculate Agent's 45%">
    The agent receives 45% of the net commission:
    ```typescript
    montoRenta = comisionNetaRenta * 0.45
    ```
  </Step>
</Steps>

## Code Implementation

From `lib/tabulador/calculations.ts:37-57`:

```typescript
export function getBaseMontos(costo: string, comision: string): BaseMontos {
  const costoNum = Number(costo || 0)
  const comisionNum = Number(comision || 0)
  
  // Sale calculations
  const comisionPrincipal = (costoNum * comisionNum) / 100
  const comisionNeta = comisionPrincipal * 0.93
  
  // Rental calculations
  const comisionPrincipalRenta = costoNum
  const comisionNetaRenta = comisionPrincipalRenta * 0.93  // 7% tax deduction
  const montoRenta = comisionNetaRenta * 0.45              // Agent gets 45%

  return {
    costoNum,
    comisionNum,
    comisionPrincipal,
    comisionNeta,
    comisionPrincipalRenta,
    comisionNetaRenta,
    montoRenta,
    montoPotencial25: comisionNeta * 0.25,
    montoPotencial20: comisionNeta * 0.2,
    montoPotencial5: comisionNeta * 0.05,
  }
}
```

## User Interface Flow

From `app/page.tsx:177-220`, the rental flow is simplified:

### Step 1: Select Operation Type

The user first selects "Renta" as the operation type:

```tsx
<button onClick={() => setTipoOperacion("renta")}>
  <p>Renta</p>
  <p>Calcular ticket directo por costo.</p>
```

### Step 2: Enter Rental Cost

The user enters only the monthly rental cost (no commission percentage needed):

```tsx
<input
  type="number"
  placeholder="Ej. 18000"
  value={costo}
  onChange={(e) => setCosto(e.target.value)}
```


## Validation Logic

The rental results are shown when the cost is valid:

```typescript
const puedeVerResultadoRenta = useMemo(() => {
  if (tipoOperacion !== "renta") return false
  return baseMontos.costoNum > 0
}, [baseMontos.costoNum, tipoOperacion])
```

**Validation requirements:**
- Operation type must be "renta"
- Cost must be greater than 0

<Note>
No scenario questions or follow-up questions are required for rentals. This makes the rental calculator much simpler than the sale calculator.
</Note>

## Tax Information

<Warning>
**7% Tax Deduction**

All rental commissions automatically deduct 7% for taxes. This is displayed to users as:

```
*Este monto ya tiene el 7% de impuestos descontado
```

The net amount after deduction is used for the 45% calculation.
</Warning>
