## Endpoint

```
POST /api/calculate
```

Calculates the total commission and its distribution between advisor, Tamivar, and broker (if applicable) based on the sale scenario.

## Request Body

<ParamField body="precioVenta" type="number" required>
  Sale price of the property
</ParamField>

<ParamField body="porcentajeComision" type="number" required>
  Commission percentage to be applied to the sale price
</ParamField>

<ParamField body="exclusiva" type="boolean" required>
  Whether the property is listed exclusively with Tamivar
</ParamField>

<ParamField body="origenCliente" type="string" required>
  Origin of the client lead. Must be one of:
  - `"asesor"` - Client came from the advisor
  - `"tamivar"` - Client came from Tamivar
  - `"broker"` - Client came from a broker
</ParamField>

<ParamField body="origenPropiedad" type="string" required>
  Origin of the property listing. Must be one of:
  - `"asesor"` - Property listed by the advisor
  - `"tamivar"` - Property listed by Tamivar
  - `"externa"` - External property listing
</ParamField>

<ParamField body="cierreEquipo" type="boolean" required>
  Whether the sale was closed as a team effort
</ParamField>

## Response

<ResponseField name="comisionTotal" type="string">
  Total commission amount calculated from the sale price and commission percentage. Returned as a decimal string with 2 decimal places.
</ResponseField>

<ResponseField name="reparto" type="object">
  Commission distribution breakdown
  
  <ResponseField name="reparto.asesor" type="object">
    Advisor's commission share
    
    <ResponseField name="reparto.asesor.porcentaje" type="number">
      Percentage of total commission allocated to the advisor
    </ResponseField>
    
    <ResponseField name="reparto.asesor.monto" type="string">
      Monetary amount allocated to the advisor (2 decimal places)
    </ResponseField>
  </ResponseField>
  
  <ResponseField name="reparto.tamivar" type="object">
    Tamivar's commission share
    
    <ResponseField name="reparto.tamivar.porcentaje" type="number">
      Percentage of total commission allocated to Tamivar
    </ResponseField>
    
    <ResponseField name="reparto.tamivar.monto" type="string">
      Monetary amount allocated to Tamivar (2 decimal places)
    </ResponseField>
  </ResponseField>
  
  <ResponseField name="reparto.broker" type="object | null">
    Broker's commission share (null if no broker involved)
    
    <ResponseField name="reparto.broker.porcentaje" type="number">
      Percentage of total commission allocated to the broker
    </ResponseField>
    
    <ResponseField name="reparto.broker.monto" type="string">
      Monetary amount allocated to the broker (2 decimal places)
    </ResponseField>
  </ResponseField>
</ResponseField>

## Commission Distribution Rules

The endpoint applies different distribution percentages based on the scenario:

| Scenario | Asesor % | Tamivar % | Broker % |
|----------|----------|-----------|----------|
| Exclusive + Advisor sells own property | 70% | 30% | - |
| Exclusive + Tamivar lead | 40% | 60% | - |
| Non-exclusive + Advisor client | 65% | 35% | - |
| Team closing | 30% | 20% | 50% (implied) |
| Base case | 50% | 50% | - |

```typescript TypeScript
interface CalculateRequest {
  precioVenta: number;
  porcentajeComision: number;
  exclusiva: boolean;
  origenCliente: "asesor" | "tamivar" | "broker";
  origenPropiedad: "asesor" | "tamivar" | "externa";
  cierreEquipo: boolean;
}

interface CalculateResponse {
  comisionTotal: string;
  reparto: {
    asesor: {
      porcentaje: number;
      monto: string;

      porcentaje: number;
      monto: string;

const request: CalculateRequest = {

  porcentajeComision: 3,

};

const response = await fetch('https://your-domain.com/api/calculate', {

    'Content-Type': 'application/json',

```

## Error Responses

### 400 Bad Request

Returned when the request body fails validation (invalid data types, missing required fields, or invalid enum values).

```json
{
  "error": "Datos inválidos"
}
```

#### Common validation errors:

- Missing required fields (`precioVenta`, `porcentajeComision`, `exclusiva`, `origenCliente`, `origenPropiedad`, `cierreEquipo`)
- Invalid data types (e.g., string instead of number)
- Invalid enum values for `origenCliente` (must be `"asesor"`, `"tamivar"`, or `"broker"`)
- Invalid enum values for `origenPropiedad` (must be `"asesor"`, `"tamivar"`, or `"externa"`)
- Invalid boolean values for `exclusiva` or `cierreEquipo`

## Implementation Details

The endpoint uses the following validation schema (Zod):

```typescript
const schema = z.object({
  precioVenta: z.number(),
  porcentajeComision: z.number(),
  exclusiva: z.boolean(),
  origenCliente: z.enum(["asesor", "tamivar", "broker"]),
  origenPropiedad: z.enum(["asesor", "tamivar", "externa"]),
  cierreEquipo: z.boolean()
})
```

All monetary calculations use the `Decimal.js` library to ensure precision and avoid floating-point errors.
